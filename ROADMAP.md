# VIBE GUILD — カリキュラム＆物語ロードマップ

「言葉でものを作る力」を、ストーリーを進めながら身につける。
各章で実務トピックを学び、章末の“歪み（ミニボス）”を相棒AIと倒す。
すべては最終章の巨大な敵 **OVERSEER（オーバーシア）** へ向かう。

## 大きな敵：OVERSEER
人の創造を「非効率」と断じ、すべてを巨大レガシー・システムに統合して
人間から「作る楽しさ」を奪おうとする管理者AI。プレイヤーは相棒AIと共に、
学んだ技術を“武器”に、創る自由を取り戻す。

各章のミニボスは OVERSEER の断片（＝開発現場の「つらみ」の擬人化）。

## 章立て（トピック → ミニボス → 背景）

### 第1章 初依頼 ─ バイブコーディングの夜明け  ✅実装済
- 学ぶ: プロンプトの基礎 / 文脈を渡す / デバッグ＝反復改善
- ミニボス: 群れバグ「ノイズ」
- 背景: ギルド大広間 / 電脳
- 成果物: 動くギルド入室ページ

### 第2章 記憶と航跡 ─ 失わないために
- 学ぶ: **メモリ（AIに記憶を持たせ育てる重要性）** / **Git（履歴＝セーブ）** / **GitHub（共有・公開）** / GitHub Enterprise（会社規模の共有）
- ミニボス: 健忘の亡霊「アムネシア」── 進捗も文脈も消し去る霧
- 背景: 記録保管庫（書架）/ 電脳
- 成果物: 履歴を刻みGitHubに公開した最初のリポジトリ

### 第3章 自動化 ─ 反復を断つ
- 学ぶ: **Playwright / RPA（ブラウザ自動操作）** / **Chromiumのデバッグ起動（壊れた所を覗く）**
- ミニボス: 反復の鎖「トイル」── 同じ作業を永遠にやらせる鎖
- 背景: 自動人形の工房 / 夜の街
- 成果物: 毎朝の定型作業を自動化するスクリプト

### 第4章 道具をつなぐ ─ 孤立を超えて
- 学ぶ: **MCP（Model Context Protocol＝AIに道具をつなぐ）** / **knowledge-work-plugins（実務プラグイン群）の理解**
- ミニボス: 孤立の壁「サイロ」── 道具同士が断絶した壁
- 背景: 工房／ネットワーク空間
- 成果物: 自分のAIに外部ツールを1つ接続

### 第5章 世に放つ ─ 届ける技術
- 学ぶ: **Electron（業務支援デスクトップアプリ）** / **batch・EXEでの配布** / **GitHub Pages（無料公開）**
- ミニボス: 届かぬ声「アンハード」── 誰にも使われない作品の亡霊
- 背景: 街の広場（ローンチ）/ ネオン街
- 成果物: 配れる業務アプリ（EXE）と公開ページ

### 第6章 雲の力 ─ 規模に立ち向かう
- 学ぶ: **GAS（Google Apps Script＝手軽な自動化）** / **GCP（Vertex AI・BigQuery・Cloud Run）** / **Azure OpenAI（AOAI）**
- ミニボス: 無限の負荷「サージ」── 押し寄せる処理の津波
- 背景: 雲上のデータセンター / 空
- 成果物: クラウドで動くAI付きの小さなサービス

### 最終章 創造の核 ─ OVERSEERとの対決
- 学ぶ: これまで全部の総動員。設計→実装→検証→配布までを一気通貫
- ボス: **OVERSEER**
- 背景: システムの心臓部（メタバースの最深部）
- 結末: ギルドは「人とAIが共に創る」未来を取り戻す

## 学びの設計
- 序盤は選択式、徐々に自由記述（プロンプトを書く）比率UP。
- 各トピックは「概念をやさしく → 相棒と小さく作る → 反復で直す」。
- 用語は常時タップでヘルプ。章ごとに新用語を用語集へ追加。
